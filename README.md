# 🚀 Auto News Hub — 半自动内容工厂

> **不会代码也能跑的内容流水线。每天10分钟，热点→文章→多平台分发。**

---

## 📌 你是做什么的？

你是**主编**，不是程序员。

每天：
1. 打开这个页面，看今天自动抓的热点
2. 挑 1-3 个你有感触的话题
3. 把话题发给我（Kimi），让我帮你写文章
4. 复制文章 → 粘贴到仓库 → 点一下发布
5. 顺手复制到 **今日头条** / **公众号**

---

## 🛠️ 每日操作流程（5步，10分钟）

### 第1步：看今天的热点
打开网站：https://hill520-git.github.io/auto-news-hub/

或者看仓库里的 `data/trends.json`。

### 第2步：挑话题
从「知乎热榜」或「微博热搜」里挑 1-3 个你感兴趣的。标准：
- 你有真实感触（失业、职场、社会、AI）
- 话题有争议或讨论度
- 你能说点"中年人视角"的实在话

### 第3步：让我写稿
把话题标题发给我，比如：
> "萝卜快跑爆火，36岁失业中年人怎么看？"

我会给你写一篇 800-1500 字的文章，语气像真人写的。

### 第4步：发布到网站
1. 在仓库里点击 `Add file` → `Create new file`
2. 文件路径：`articles/YYYY-MM-DD-文章标题.md`
3. 把我写的内容粘贴进去
4. 点 `Commit changes`
5. 网站会自动更新（2分钟内）

### 第5步：多平台分发
- **今日头条**：复制全文，粘贴发布（有阅读就有收益）
- **公众号**：稍微润色，加几句个人感受，发布
- **知乎想法/回答**：把文章拆成 3-5 段回答相关问题

## ⚙️ 启用 GitHub Pages（1分钟搞定）

1. 打开仓库页面：https://github.com/hill520-git/auto-news-hub
2. 点顶部 `Settings`
3. 左侧菜单找到 `Pages`
4. **Source** 选择 `Deploy from a branch`
5. **Branch** 选择 `main`，文件夹选 `/(root)`
6. 点 `Save`
7. 等 1-2 分钟，访问 https://hill520-git.github.io/auto-news-hub/

## ❓ 常见问题

**Q: 热点每天几点更新？**
A: 北京时间每天早上 8:00 自动更新。也可以手动触发：进入 Actions → Daily Trends Fetch → Run workflow。

**Q: 网站打不开？**
A: 首次启用 GitHub Pages 后需要等 1-5 分钟。如果超过10分钟还没好，检查一下 Settings → Pages 里的配置。

**Q: 文章怎么排版好看？**
A: 用 Markdown 格式，我会帮你写好。你也可以在发布前让我帮你排版。

---
